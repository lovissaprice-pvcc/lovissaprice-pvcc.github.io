Chapter 5: Programming Algorithms and Software Testing


Phases of Computer Program Developement:

1 Understand the problem
2 Design an algorithm to solve the problem
3 Write the program code in appropriate language based on the algorithm
4 Enter the program and save
5 Execute the program
6 Test the program, find errors and debug, then reset
7 Document and evaluate the program

Programming Structures

  • Sequential Structure: top-down execution

  • Decision structure: if/then/else

  • Iterative (looping):
    1. While loop: continue looping as long as something is true
    2. Repeat/until loop: continue looping until something is true
    3. For loop: loop a given number of times

  • Recursive structures:
    1. Repeat a set of instructions as a subtask of itself
    2. Repetition continues until:
      • a degenerative case(also "base case"), which is a degenerative case, is met

Two classic programming/database problems

  • Sorting: descending order/ascending order
    1. Sorting numbers
    2. Sorting strings (in accordance with ASCII values)
  • Searching: find specific data that meet a certain criteria
    1. In database:
      1. Filters: temporary search
      2. Queries: search criteria are saved so they don't have to be recreated
  • Two popular search techniques:
    1. Sequential search: search from top to bottom or bottom to top, one item at a time
    2. Binary search:
      • only works if data is SORTED first
      • Start in the middle, then go to half of that, etc.

Two Classic Program Goals:

  1. Efficiency: program takes minimal time and/or resources
  2. Correctness: program runs in expected ways and works for all expected conditions

Software Testing and Verification

Types of Tests:

Term Definition
Acceptance testing Verifying whether the whole system works as intended
Integration testing Ensuring that software components or functions operate together
Unit testing Validating that each software unit runs as expected. A unit is the smallest testable component of an application
Functional testing Checking functions by emulating business scenarios, based on functional requirements. Black-box testing is a common way to verify functions
Performance testing Testing how the software runs under different workloads. Load testing, for example, is used to evaluate performance under real-life load conditions
Regression testing Checking whether new features break or degrade functionality
Stress testing Testing how much strain the system can take before it fails. Stress testing is considered to be a type of non-functional testing
Usability testing Validating how well a customer can use a system or web application to complete a task

  • Software testing is done to detect:
    1. Architectural flaws
    2. Poor design decisions
    3. Invalid or incorrect functionality
    4. Security vulnerabilities
    5. Scalability issues


  • Software testing best practices:
    1. Continuous testing: project teams test each build as it becomes available
    2. Configuration management: organizations centrally maintain test assets and track what software builds to test
    3. Service virtualization: simulates the services and systems that are missing or not yet completed, enabling teams to reduce dependencies and test sooner
    4. Defect or "bug tracking": automated tools allow teams to track defects, measure their scope and impact, and uncover related issues
    5. Metrics and reporting: teams quickly see the overall health of a project and can monitor relationships between test, development and other project elements
