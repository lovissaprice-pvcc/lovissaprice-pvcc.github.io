Chapter 7: Software Engineering

Term Definition
Software Engineering Seeks principles to guide the development of large, complex software systems
Integrated Developement Environment (IDE) Combine tools for developing software (editors, compilers, debugging tools, and so on) into a single, integrated package
Association for Computing Machinery (ACM)
  1. Purpose of ACM
  2. Benefits of ACM membership
  3. Student ACM membership
  1. Dedicated to advancing the arts, sciences, and applications of information technology
  2. It offers publications, conferences, and career resources as well as networking opportunities
  3. Only $19
Institute of Electrical and Electronics Engineers (IEEE)
  1. Purpose of IEEE
  2. Benefits of IEEE membership
  3. Student IEEE membership
  1. Advancing technology for the benefit of humanity
  2. Mostly networking, including special events and online communication with other people in the field
  3. Only $16
Software Life Cycle: Developement Phase
  1. Requiremenet Analysis
  2. Design
  3. Implementation
  4. Testing
  1. Specifies what services the proposed system will provide, to identify any conditions (time constraints, security, and so on) on those services, and to define how the outside world will interact with the system
  2. creating a plan for the construction of the proposed system
  3. Writing of programs, creation of data files, and development of databases
  4. Turns the previous phases into: “requirements analysis and confirmation,” “design and validation,” and “implementation and testing.”
Modular Implementation Separating the functionality of a program into independent, interchangeable modules
Unified Modeling Language (UML) standardized modeling language consisting of an integrated set of diagrams
Use Case A usage scenario for a piece of software
Use Case Diagram Captures the image of the proposed system from the user’s point of view
Data Relationship:
  1. One-to-one
  2. One-to-many
  3. Many-to-many
  1. Each patient is associated with only one room and each room is associated with only one patient
  2. One physician is associated with many patients and each patient is associated with one (primary) physician
  3. Each physician could be associated with many patients and each patient could be associated with many physicians
Design Pattern A predeveloped model for solving a recurring problem in software design
Software Quality Assurance (SQA) Overseeing and enforce the quality control systems adopted by software organizations
Software Testing:
  1. Glass box testing
  2. Black box testing
  3. Beta testing
  4. Alpha testing
  1. The software tester is aware of the interior structure of the software and uses this knowledge when designing the test
  2. Performed from the user’s point of view because the tester has no nowledge of the software's functionality
  3. A preliminary version of the software is given to a segment of the intended audience with the goal of learning how the software performs in real-life situations
  4. Beta testing performed at the developer’s site
Documentation:
  1. User documentation
  2. System documentation
  3. Technical documentation
  1. Explains the features of the software and describe how to use them
  2. Describes the software’s internal composition so that the software can be maintained later in its life cycle
  3. Describes how a software system should be installed and serviced
Intellectual Property Law
  1. Software License
  2. Software Disclaimer
  3. Best defenses against software liability claims
  1. The rights of the developer
  2. State the limitations of the developers' liability
  3. Apply sound software engineering principles during the software’s development, to use a level of care compatible with the software’s application, and to produce and maintain records that validate these endeavors
