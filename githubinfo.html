Q & A

  • Q. What is a version control system?
  • A. Tools that help manage changes to source code over time.

  • Q. What is a Git
  • A. A distributed verion control system.

  • Q. What is GitHub?
  • A. A platform that alows developers to create, store, manage, and share code.

  • Q. What is the difference between Git and GitHub?
  • A. Git is a version control system while GitHub is a cloud-based hosting service that lets you manage Git repositories

  • Q. Who started GitHub and how was it started
  • A. Founded by Tom Preston-Werner, Chris Wanstrath, and PJ Hyett in 2008 when they collaborated on a fun weekend project

  • Q. What cmpany owns it now?
  • A. Microsoft Corp

  • Q. How much does a GitHub account cost?
  • A. $0

  • Q. What is the Octocat?
  • A. It is an eight-legged cat that is GitHub's logo.


Git/GitHub Terms:

  1. Repository: a central location in which data is stored and managed.

  2. Commit: the process of submitting changes made to a code repository.

  3. Fork: a new repository that shares code and visibility settings with the original “upstream” repository.

  4. Push: uploading code from a branch in your local source code repository to a branch in the remote repository.

  5. Pull requests: a proposal to merge a set of changes from one branch into another.

  6. Workflows: a sequence of tasks that processes a set of data through a specific path from initiation to completion.

  7. Issue: caught non-fatal exceptions or application not responding (ANR) issues.

  8. Raw button: opens the file in a raw form, meaning that any HTML formatting disappears.

  9. Blame button: examines file contents line-by-line and identifies the author and time of the modifications.
